Phase 3 도메인 & DNS (Cloudflare) ~30분 + DNS 전파 대기
목표 — 도메인 구매 + Cloudflare 네임서버 + Origin Cert 발급까지.
3-1Cloudflare Registrar 에서 도메인 구매

Cloudflare Dashboard → Domain Registration → 검색 → 구매.

  • jsonjeong.com — $10/년 수준
  • 구매 시 자동으로 Cloudflare 네임서버 세팅
GoDaddy/Namecheap 등 외부에서 산 도메인이면 네임서버를 Cloudflare 로 변경 필요 (전파 24-48시간).
3-2DNS 레코드 추가 (드롭릿 IP 가 있다면 - 임시 placeholder 도 OK)
# Cloudflare Dashboard → DNS → Records
A     jsonjeong.com         <droplet-IP>     Proxied
A     www.jsonjeong.com     <droplet-IP>     Proxied
A     newsletter.jsonjeong.com <droplet-IP>  Proxied

드롭릿 IP 는 Phase 5 에서 확정. 일단 placeholder 로 만들고 나중에 수정해도 됨.

3-3SSL/TLS 모드 = Full (strict)

Cloudflare Dashboard → SSL/TLS → Overview → Full (strict) 선택.

Flexible 모드는 절대 사용 금지. 클라이언트 ↔ Cloudflare 만 암호화하고 Cloudflare ↔ Origin 은 평문이라 MITM 위험.
3-4Cloudflare Origin Certificate 생성

SSL/TLS → Origin Server → Create Certificate.

  • Hostnames: *.jsonjeong.com, jsonjeong.com (와일드카드 포함)
  • Validity: 15 years (최대)
  • 생성된 certificate.pem 과 private_key.pem 을 다운로드
# 인증서 파일을 레포 밖에 안전하게 보관
mkdir -p ~/.config/jsonjeong-cert
mv ~/Downloads/certificate.pem ~/.config/jsonjeong-cert/origin.crt
mv ~/Downloads/private_key.pem ~/.config/jsonjeong-cert/origin.key
chmod 600 ~/.config/jsonjeong-cert/*
초기엔 Let's Encrypt 로 시작했다가, Cloudflare proxied 모드에서 Let's Encrypt 인증서 발급 challenge 가 어려워서 Origin Cert 로 전환. 처음부터 Origin Cert 가 정답.

목표 — 인덱스 사이트의 핵심 페이지(대시보드 + 뉴스레터 + 인증) 구현.
4-1Rails 8 내장 인증 generator

4-3DashboardController + 서비스 카드 그리드

app/controllers/dashboard_controller.rb 작성. SERVICES 상수에 각 SaaS 카드 정의. 외부 도메인 URL 로 link.

SERVICES = [
  Service.new(name: "SnapDeck", path: "https://snapdeck.com", ...),
  Service.new(name: "SnapTeam", path: "https://snapteam.com", ...),

새 서비스 추가는 이 배열에 한 줄 추가하는 것. 별도 레포는 따로 만들고 여기선 카드만.
4-4NewsletterController + 서브도메인 라우팅
config/routes.rb 에 서브도메인 제약 추가: 
constraints subdomain: "newsletter" do
  root "newsletter#show"
  post "/" => "newsletter#subscribe"
end

constraints subdomain: /\A(www)?\z/ do
  root "dashboard#index"
  post "newsletter", to: "newsletter#subscribe"
end
로컬 개발에선 lvh.me 사용 (newsletter.lvh.me:3000 이 자동으로 127.0.0.1 로 풀림).
4-5BeehiivClient 서비스
app/services/beehiiv_client.rb 작성. Net::HTTP 로 직접 호출. API 키는 Rails encrypted credentials 에서 읽음: 
bin/rails credentials:edit

# beehiiv:
#   api_key: bh_...

Phase 5 서버 프로비저닝 (DigitalOcean) ~30분
목표 — SSH 로 접속되는 Ubuntu droplet 1개.
5-1Droplet 생성
DigitalOcean → Create → Droplet:
  • Image: Ubuntu 24.04 LTS
  • Plan: Basic / Regular / $6/월 (1GB RAM, 25GB SSD)
  • Region: Singapore 또는 San Francisco (한국에서 가까운 곳)
  • Authentication: SSH key (계정 SSH key 등록 후 선택)

생성 후 IP 주소 메모. Cloudflare DNS A 레코드도 이 IP 로 업데이트.

5-2SSH 접속 확인
ssh root@<droplet-IP>

Phase 6 배포 인프라 (Kamal + GHCR) ~1-2시간
목표 — bin/kamal deploy 한 줄로 배포되는 상태.
6-1GHCR 용 GitHub PAT 발급
GitHub → Settings → Developer settings → Personal access tokens (classic):
  • Scopes: write:packages, read:packages
  • Note: jsonjeong-kamal
# 환경변수로 보관 (~/.zshrc 또는 .envrc)
export KAMAL_REGISTRY_PASSWORD="ghp_..."
PAT 는 절대 git 커밋하지 말 것. .kamal/secrets 안에서는 $KAMAL_REGISTRY_PASSWORD 로만 참조.
6-2config/deploy.yml 작성
service: jsonjeong
image: snapplug/jsonjeong

servers:
  web:
    - <droplet-IP>

proxy:
  ssl:
    certificate_pem: CERTIFICATE_PEM
    private_key_pem: PRIVATE_KEY_PEM
  hosts:
    - jsonjeong.com
    - www.jsonjeong.com
    - newsletter.jsonjeong.com
  app_port: 80

registry:
  server: ghcr.io
  username: SnapPlug
  password:
    - KAMAL_REGISTRY_PASSWORD

builder:
  arch: amd64

env:
  secret:
    - RAILS_MASTER_KEY

ssh:
  user: root

volumes:
  - "jsonjeong_storage:/rails/storage"

asset_path: /rails/public/assets
volumes 설정을 빼먹으면 매 배포마다 SQLite DB 가 날아감. 가장 중요한 한 줄.
6-3.kamal/secrets 작성 (커밋 안전 버전)
# .kamal/secrets — git 커밋 OK (플레인텍스트 시크릿 없음)
RAILS_MASTER_KEY=$(cat config/master.key)
KAMAL_REGISTRY_PASSWORD=$KAMAL_REGISTRY_PASSWORD
CERTIFICATE_PEM=$(cat ~/.config/jsonjeong-cert/origin.crt)
PRIVATE_KEY_PEM=$(cat ~/.config/jsonjeong-cert/origin.key)
실제 시크릿은 ENV 또는 레포 밖 파일에서 읽음. .kamal/secrets 자체는 ENV 이름과 외부 경로만 적힘.
6-4.gitignore 점검
다음 항목이 무시되는지 반드시 확인: 
/.env*
/config/*.key
/storage/*
/log/*
/tmp/*
git check-ignore -v config/master.key 가 .gitignore 룰 출력

7-2실제 겪은 함정 — TLS 전환 이슈
증상: Cloudflare proxied + Let's Encrypt 조합에서 인증서 발급 실패 (ACME challenge 가 Cloudflare 에서 막힘).
해결: Let's Encrypt 포기 → Cloudflare Origin Cert 로 전환 (Phase 3-4 에서 미리 준비).
커밋 흔적: 0c17a00 deploy: switch to Cloudflare Origin Cert for Full(strict) SSL
