3. 트레이드오프 비교

관점 Next + Supabase + Vercel Rails 모놀리스
시작 속도 클릭 몇 번이면 끝 (가입 → 프로젝트 생성 → 배포) Docker / Kamal 학습 곡선 1~2일
운영 부담 0 (전부 매니지드) VPS 본인 관리 (백업, 보안 패치)
스케일링 트래픽 폭증 자동 대응 (서버리스) VPS 수직 확장 (대부분 충분, 수만 동접까지)
월 비용 (소규모) Free tier → 트래픽 늘면 가파르게 상승 ($50~수백 $) $6/월 고정 (DO droplet)
월 비용 (중규모) Supabase Pro $25 + Vercel Pro $20 + DB row 비용 + 함수 호출 비용 $6~$24 (droplet 사이즈 업)
벤더 락인 강함 — Supabase 떠나기 어려움, Server Actions 는 Vercel 종속 없음 — Postgres/MySQL 어디든, VPS 어디든
데이터 주권 미국 회사 서버 (의료/금융/공공 분야는 이슈) 본인 서버 위치 선택 가능
인지 비용 3개 서비스 + JS 생태계 (npm 패키지 수십~수백) 1개 프레임워크 (관례 학습)
AI 코딩 적합도 패턴 분산 → AI 가 잘 모르면 혼란 가능 관례 강함 → AI 가 학습한 패턴 그대로 통함
실시간 / 협업 기능 Supabase Realtime 강력 (Postgres 변경 → 자동 푸시) Action Cable 직접 구현 (코드량 많음)
모바일 앱 백엔드로 Supabase REST/RPC 즉시 사용 가능 JSON API 컨트롤러 직접 작성

4. 언제 어떤 걸 골라야 하나

Next + Supabase + Vercel 이 유리한 경우

  • React 생태계 의존이 명확 (기존 디자인 시스템, 컴포넌트 라이브러리)
  • 모바일 앱 동시 개발 (Supabase 가 백엔드 역할)
  • 실시간 협업 기능이 핵심 (Figma, Linear 류)
  • 마케팅/랜딩 위주 + 잦은 PR 프리뷰가 필요
  • 운영 인력 0, 인프라 신경 쓰기 싫음
  • 해외 트래픽 글로벌 분산이 중요

Rails 모놀리스가 유리한 경우

  • 1인 메이커, 풀스택 단일 언어로 빠르게 (jsonjeong 같은 경우)
  • CRUD 비중이 높은 SaaS / 어드민 도구
  • 벤더 락인 회피, 데이터 직접 소유
  • 비용을 절대 통제하고 싶음 ($6 고정)
  • 마크 루처럼 여러 작은 서비스를 빠르게 찍어내는 패턴
  • AI 코딩 시 관례 강한 프레임워크 선호
핵심 인사이트 — "어떤 스택이 좋은가" 가 아니라 "어떤 자리들을 몇 개의 박스로 합칠 것인가" 가 진짜 의사결정입니다. Next+Supabase+Vercel 은 3개 박스로, Rails 모놀리스는 1개 박스로 같은 자리를 채웁니다. 박스가 많아질수록 유연성과 매니지드 편의성이 늘고, 박스가 적어질수록 비용·인지 부담·벤더 락인이 줄어듭니다.

5. 우리(jsonjeong.com) 의 선택 근거

  • 1인 메이커 — 박스를 줄여야 인지 부담 최소화
  • 여러 SaaS 를 별도 레포로 찍어내는 마크 루 패턴 — 박스 적은 템플릿이 복붙하기 쉬움
  • AI 코딩 적합도 — Rails Omakase 관례가 강해 Claude Code 가 패턴을 잘 잡음
  • 비용 통제 — $6 droplet 고정, 매출 0 단계에서 부담 없음
  • 벤더 락인 회피 — 미국 SaaS 의존 0, 매각 가능 (마크 루 매각 사례)
