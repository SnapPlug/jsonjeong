웹 서비스에 필요한 "자리"는 보통 12~15개 정도 — 어떤 회사도 이 자리들을 채울 뿐이고, 다만 어떤 자리는 매니지드 서비스(Vercel, Supabase) 가 통째로 가져가고, 어떤 자리는 우리처럼 Rails 모놀리스가 통째로 가져갑니다.

핵심 의사결정 = "이 자리들을 몇 개의 박스로 합칠 것인가" — 1인 메이커는 박스를 최소화하고 (Rails 8 모놀리스), 대규모 조직은 박스를 분리합니다 (마이크로서비스). 우리는 박스를 최대한 줄여서 외부 의존성 0(Redis 0, Node 0) 으로 갔습니다.
